## Technology Rules

### Frontend (web/)
- Next.js 14 App Router ONLY — no pages/ directory
- Vercel AI SDK for all chat streaming
- Tailwind CSS with ROAR design tokens from tokens.css
- ALL colors via CSS custom properties — NEVER hardcode hex values
- Dark mode via .dark class on <html> — class-based, not media query
- Polling for live chat: GET /cases/:id/messages every 4000ms (CHAT_POLL_INTERVAL_MS)
- No localStorage for auth — httpOnly cookie or in-memory JWT
- TypeScript strict mode

### Backend (api/)
- FastAPI with async endpoints throughout — no sync def for DB operations
- SQLAlchemy async ORM + asyncpg driver
- python-jose for JWT, passlib[bcrypt] for password hashing
- ALL DB access through FastAPI — n8n never touches DB directly
- /internal/sources/* endpoints protected by X-Webhook-Secret header
- /webhooks/* endpoints protected by X-Webhook-Secret header
- Status transitions validated on every PATCH /cases/:id — invalid = HTTP 422
- chat_messages table is APPEND-ONLY — no DELETE or UPDATE endpoints

### Database (db/)
- PostgreSQL on Railway
- Migrations in db/migrations/ — numbered 001, 002, etc.
- Seeds in db/seed/ — run in order: 001_policies → 002_users → 003_scenario_a → 004_scenario_b
- UUIDs for all primary keys (gen_random_uuid())
- All timestamps as timestamptz

### n8n Workflows
- 6 workflows, one per agent — see ROAR_n8n_Spec_v2.1.md
- WF2 calls FastAPI /internal/sources/* — NEVER queries DB directly
- All agents use GPT-4o-mini with JSON output mode
- Temperature: 0.0 for Triage (WF3) and Case Report (WF6), 0.2 for others
- Always implement fallback on every LLM call — never throw uncaught on AI failure

- Primary brand color token: --color-primary (#D4581A light / #E8862E dark)
- Font: Inter (sans) + JetBrains Mono (mono) via next/font
- Border radius: --radius-card (12px), --radius-btn (8px), --radius-pill (999px)
- ALL component specs in ROAR_DesignGuidelines_v1.md §5 — follow exactly
- CaseStatusPill colors must match status exactly — see BRL §3.1 status definitions
- Never use bold weight 600 or 700 in body text — only headings and labels

## Code Quality Rules
- No console.log in production code — use proper error handling
- Every FastAPI endpoint must have a docstring
- Every async function must have proper error handling with try/except
- n8n workflow Code nodes must have try/catch on every JSON.parse
- TypeScript: no `any` types — define proper interfaces in web/types/index.ts
- Environment variables: never hardcode, always from .env — validate on startup

## File Naming
- React components: PascalCase (CaseCard.tsx)
- Utilities and hooks: camelCase (usePolling.ts)
- API routers: snake_case (cases.py)
- DB migrations: 001_description.sql
- n8n workflows: 01_workflow_name.json

## What NOT To Do
- Do not use Supabase client libraries — we use PostgreSQL directly via SQLAlchemy
- Do not use Supabase Auth — we use FastAPI JWT
- Do not use WebSockets for chat — we use polling
- Do not use pages/ directory in Next.js
- Do not hardcode any color values in components
- Do not skip status transition validation in FastAPI
- Do not allow n8n to query the database directly
- Do not make chat_messages deletable or editable
- Do not implement settings screen — out of scope
- Do not implement analytics dashboard — out of scope
